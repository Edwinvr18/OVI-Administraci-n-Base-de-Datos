Esquema y planteamiento de bases de datos relacional.

Las bases de datos son herramientas muy útiles que nos ayudan a entender y agrupar mejor ciertos datos que tenemos, para este caso una base de datos relacional es una recopilación de elementos de datos con relaciones predefinidas entre ellos. Estas están estructuradas de acuerdo con cómo se desee enlazar la información, es decir por sus atributos o datos que contienen cada tabla de una base de datos. Este modelo de organización y gestión de bases de datos consistente generalmente en el almacenamiento de datos en tablas compuestas por filas, o tuplas, y columnas o campos. Se distingue de otros modelos, como el jerárquico, ya que llega a ser mucho más comprensible para el usuario inexperto.
Entre los conceptos que rodean este término están por ejemplo:
Tabla: es el nombre que recibe cada una de las relaciones que se establecen entre los datos almacenados, dentro de estas tablas los datos están agrupados en columnas y filas.
Claves: elementos que impiden la duplicidad de datos.

Existen dos grandes tipos de claves: las claves primarias y las secundarias o externas.

Claves primarias: Es el tipo de relación que se ha definido en la tabla. Pueden añadirse otros atributos específicos y propios.
Claves externas o secundarias: Se definen para cada una de las claves primarias establecidas para los elementos o entidades de una relación.
